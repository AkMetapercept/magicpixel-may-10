# Add a New Tag (For Server-side)

To add a new server-side tag, perform the following steps:

1.	Select **Tags** from the left navigation panel.

2.	Click **Add Tag -> For Server** on the top-right corner of the screen.

![addanewtagserver](/images/tagsimages/addanewtagserver.png)

3.	Select the **Provider Name** or enter the Provider name from the list to proceed further.

![addanewtagserver](/images/tagsimages/addanewtagserverprovider.png)

**Step Result:** The New tag screen appears.

![addanewtagserver](/images/tagsimages/addanewtagserverscreenprovider.png)

4. Select **Trigger**. 

You can select either the **User-defined** Triggers or **Default** triggers using this option and click **Done**.

**Step Result:** The tag is triggered for the respective trigger data.

![addanewtagserver](/images/tagsimages/choosetriggers.png)

## Set Qualification Criteria

Qualification Criteria of a tag are conditions/triggers that inform Ingest Labs when to trigger a particular tag. 

In this step, you can add the Qualification Criteria for the tag in the **Qualification Criteria required** screen. You can add multiple Qualification Criteria and these criteria are layered.

![addanewtagserver](/images/tagsimages/addnewtagserverqualificationcriteria.png)

1. Click **Add Qualification Criteria**.

2. Select the **Data Elements** tab for the data elements to be set for the Qualification criteria.

![addanewtagserver](/images/tagsimages/choosedataelements.png)

## Set Tag Info

This is the step where you set a name and validity period for the tag. The default validity period is one year for any new tag that is created and can be set to any desired date. (To understand tag validity or expiry, see the section “Set the Validity period and Tag Expiry”:

**Name:** The Tag name.

**Validity Period:** Denotes the validity period of the tag including Start Date and End Date.

![addanewtagserver](/images/tagsimages/taginfovalidityperiod.png)

## Set the Validity Period and Tag Expiry

While creating a tag on Ingest Labs, you can set the validity period of a particular tag and define tag expiry. The default validity period is one year for any new tag that is created and can be set to any desired date. When the expiry date is closer, you will receive reminder e-mail notification about the expiry of the tag. If no action is taken, the tag stops triggering post the set validity period. You can find all expired tags in the “Paused State” in the Publish section and you can remove the expired tags to reduce the size of your SDK.

## Set Tag Query Parameters

The Query parameters include the set of dynamic values that are included along with the Base URL while executing the query. These parameters appear in the Attributes section as well along with the values.
 
## Set Provider Attributes

The Provider is a mandatory setting while adding a new tag. You can also create a new (custom) provider using the Providers option on the left navigation panel. You can add any number of Provider attributes using the Key-Value pair and configure the settings using the Settings option.

For a particular Provider attribute, you must set the value either from the Data Elements or Transformers.

![addanewtagserver](/images/tagsimages/setproviderattributes.png)

After selecting either from the Data Element or Transformer, the value appears in the Value column.

![addanewtagserver](/images/tagsimages/providerattributesvalueserver.png)

**Add Attributes**

1. To add Attributes, select the **Add Attributes** and set the following values.

![addanewtagserver](/images/tagsimages/addattributesserver.png)

**Name**: Set the Attribute name. For example: visitor_id, user_id, and so on.

**Value**: Set the value either for the Data Element or Transformer.

**JSON**: Enter the Attribute name and value, and configure the Settings.

- Mandatory - Denotes the value to be mandatory.
- Ignore If Empty - Ignore the field if empty.
- Hash (SHA256) - denotes the value in the HASH format.

![addanewtagserver](/images/tagsimages/addattributesserverjson.png)

**Static**: Denotes the static Value that is known.

**Attribute Settings**:

2. Enter the **Attribute Settings**.

**Tool-tip!** Select the **Mandatory** check box to display the **Fatal** check box.

![addanewtagserver](/images/tagsimages/serverattributesettings.png)

3. Set the Data Type Conversion using the following options.

| Data Type Conversion      | Description |
| ----------- | ----------- |
| No Conversion     | Indicates that there is no conversion required for the data type.  |
| Convert to Boolean   | Converts the input data to Boolean.        |
| Convert to Number | Converts the input data to Number. |
| Convert to Array | Converts the input data to Array. |
| Convert to Object Spread | Converts the JSON attributes under a parent level attribute to the root level attributes. |

4. Select one the following Transformer types and click **Confirm**.

| Transformer Type      | Description |
| ----------- | ----------- |
| URL encode     | Converts the URL into the encrypted format.  |
| HASH (SHA256)   | The value is converted to HASH (SHA256) format. |
| Ignore if empty | May be ignored if the value is empty. |

**Step Result:** The Provider Attribute is created.

To delete the Provider Attribute, select the **Delete** icon at the right-hand corner of the screen.

## Set Tag Attributes

To add Tag Attributes, click **Add Attribute**.

![addanewtagserver](/images/tagsimages/addattributesserver.png)

**Name**: Set the Attribute name. For example: visitor_id, user_id, and so on.

**Value**: Set the value either for the Data Element or Transformer.

**JSON**: Enter the Attribute name and value, and configure the Settings.

- Mandatory - Denotes the value to be mandatory.

- Ignore If Empty - Ignore the field if empty.

- Hash (SHA256) - denotes the value in the HASH format.

**Static**: Denotes the static Value that is known.

![addanewtagserver](/images/tagsimages/addattributesserverjson.png)

**Attribute Settings**: 

1. Enter the **Attribute Settings**.

**Tool-tip!** Select the **Mandatory** check box to display the **Fatal** check box.

![addanewtagserver](/images/tagsimages/serverattributesettings.png)

2. Set the Data Type Conversion using the following options.

| Data Type Conversion      | Description |
| ----------- | ----------- |
| No Conversion     | Indicates that there is no conversion required for the data type. |
| Convert to Boolean   | Converts the input data to Boolean.        |
| Convert to Number | Converts the input data to Number. |
| Convert to Array | Converts the input data to Array. |
| Convert to Object Spread | Converts the JSON attributes under a parent level attribute to the root level attributes.|

3. Select one of the following Transformer types and click **Confirm**.

| Transformer Type      | Description |
| ----------- | ----------- |
| URL encode     |  Converts the URL into the encrypted format.    |
| HASH (SHA256)   | The value is converted to HASH (SHA256) format. |
| Ignore if empty | May be ignored if the value is empty. |

**Step Result:** The Tag Attribute is created.

4. Enable **Retry Configuration**.

![addanewtagserver](/images/tagsimages/retryconfiguration.png)

5. Enable **Delay Configuration**.

![addanewtagserver](/images/tagsimages/delayconfiguration.png)

6. Click **Save** to save the settings.

**Step Result:** The new tag is created.
